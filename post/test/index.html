yarn add webpack@4.44.2 webpack-cli@3.3.12 webpack-dev-server@3.11.2 -D

  • 建立public/index.html文件作为模板文件，嵌入vue cdn

// vue3版本

vue2版本需要降低vue-loader版本，上述配置支持vue3。测试项目可以自己搭建webpack，生产环境建议使用脚手架工具。

配置webpack.config.js文件

// node提供,可以将相对路径转换为绝对路径 __dirname为当前文件位置路径
const path = require('path'); 
const HtmlWebpackPlugin = require('html-webpack-plugin');
const {VueLoaderPlugin} = require('vue-loader');

module.exports = {
    mode: 'development',
    entry: path.resolve(__dirname, 'src/main.js'),
    output: {
        path: path.resolve(__dirname, 'dist'),
        filename: "[name][hash].js"
    },
    externals: {
        'vue': 'Vue'
    },
    devtool: "source-map",
    module: {
        rules: [{
            test: /\.vue$/,
            loader: "vue-loader"
        }]
    },
    plugins: [
        new VueLoaderPlugin(),
        new HtmlWebpackPlugin({template: path.resolve(__dirname, 'public/index.html')}),
    ]
}

（四）初识vue

vue核心思想

vue核心思想是对模板语法进行编译，渲染DOM。 vue将数据与DOM进行关联，并建立响应式关系。所谓响应式，就是数据改变，视图改变。 数据 => viewModel核心库 => 视图。

  • template：组件模板
  • script: 组件逻辑模块。组件逻辑的本质是一个对象，其中含有了很多特定的属性。
  • style: 组件样式

双向绑定与react的区别

vue支持视图驱动数据发生改变。React不支持，React必须使用事件驱动更改state从而触发视图的更新。 如果不看v-model，vue也可以被称为单向数据流。vue完成了数据双向绑定的机制，使我们的关注点可以完全专注于业务逻辑层，视图层交给viewModel库完成数据绑定、渲染和更新。

vue简单使用示例

<!doctype html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie=edge">
    <title>Document</title>
</head>
<body>
    <div id="root">
        <div>{{title}}</div>
        <!-- v-on:click 绑定点击事件，会被转换为addEventListener。可简写:@click=handleAddLike()-->
        <div>Like: {{like}} <button v-on:click="handleAddLike">点赞</button></div>
        <div>
            <span v-if="isLogin">已登录</span>
            <span v-else>未登录</span>
            <button @click="toggleLoginStatus">切换登录状态</button>
        </div>

        <!-- v-bind 绑定属性，引号内部看作变量，vue 会对其进行解析 -->
        <p :title="title">{{title}}</p>

        <!-- v-model 双向绑定表单控件 -->
        <div>
            {{inputVal}}
            <input type="text" v-model="inputVal"/>
        </div>

        <!-- v-for 循环 -->
        <!-- key of list 对象使用-->
        <!-- key in list 数组使用-->
        <ul>
            <li v-for="item of list" :key="item">{{item}}</li>
        </ul>

        <span v-for="key in obj" :key="key">{{key}}</span>
    </div>
    <script src="https://cdn.jsdelivr.net/npm/vue@3.1.2/dist/vue.global.js"></script>
</body>
</html>

const Acticle = {
    data() {
        return {
            title: 'This is a title',
            like: 0,
            isLogin: false,
            inputVal: '',
            list: [1, 2, 3, 4, 5],
            obj: {name: 'lily', sex: '男', age: 12}
        }
    },
    methods: {
        handleAddLike() {
            this.like += 1;
        },
        toggleLoginStatus() {
            this.isLogin = !this.isLogin;
        }
    }
}

Vue.createApp(Acticle).mount('#root');

Vue组件化构建

vue利用ES模块化，来完成vue的组件系统构建。 组件化即抽象成小型、独立、可预先定义配置的、可复用的组件。

  • 小型：页面的构成拆分成一个一个的小单元独立开发。
  • 独立：每一个小单元尽可能都独立开发。
  • 预先定义：小单元都可以先定义好，在需要的时候导入使用。
  • 预先配置：小单元可以接收一些在使用的时候需要的配置。
  • 可复用：一个小单元可以在多个地方使用，

可复用性要适当考量，组件最大的作用是独立开发、预先配置。目的是为了更好的维护和扩展，有些组件的确是不需要复用的，可配置性越高，功能就越强。

组件嵌套形成组件树。

应用实例、组件实例与根组件实例

应用组件

Vue.createApp()返回一个应用实例。 实例上暴露了很多方法，且大多数方法都会返回应用实例本身:

  • component 注册组件
  • directive 注册指令
  • filter 注册过滤器
  • use 使用插件
  • …
const app = Vue.createApp({}); // application 应用

通过应用实例注册全局组件

应用实例主要用来注册全局组件。

        </div>
    `
}).directive('toLowerCase', {
    mounted(el) {
        el.addEventListener('click', function () {
            this.innerText = this.innerText.toLowerCase();
        }, false)
    }
});

根组件实例

根组件的本质就是一个对象。

  • 根组件： createApp({})执行的时候需要一个根组件，根组件是vue渲染的起点。
  • 根元素：createApp()创建应用实例时需要挂载到一个HTML根元素上。

app.mount()返回根组件实例（vm），而不是应用实例。vm即viewModel，MVVM => vm Vue 不是一个完整的 MVVM 模型，只是参考了 MVVM 模型。

// index.html
<div id="app"></div>

const RootComponent = {
  data () {
    return {
      a: 1,
      b: 2,
      total: 0
    }
  },
  mounted () {
  	this.plus();
  },
  methods: {
    plus () {
      this.total = this.a + this.b;
    }
  },
  template: `<h1>{{ a }} + {{ b }} = {{ total }}</h1>`
}

const app = Vue.createApp(RootComponent);
const vm = app.mount('#app');

组件实例

每个组件都有自己的组件实例，一个应用中所有的组件都共享一个应用实例。 无论是根组件还是应用内其他的组件，配置选项、组件行为都是一样的。 组件实例可以添加一些属性 property。

认识及实现MVC

MVC

MVC实际上是MVVVM的雏形，MVVM解决了驱动不内聚、不集中，解决了视图和模型完全隔离的桩体

后端MVC

  • Model（M）数据模型层。操作数据库（对数据进行增删改查的操作）
  • View（V）视图层。显示视图或者模板（服务端渲染时存在V）
  • Controller（C）控制器，逻辑层。将数据与视图关联挂载，有一些基础逻辑操作。也可以充当API层，前端请求接口时对应的就是后端的controller层。

后端渲染 前端异步请求URL（controller中的一个方法），通过Model层的方法，操作数据库获取数据，返回给控制器方法，响应回前端。

前端MVC

  • Model(M)数据层。管理视图所需要数据，数据关联，数据与视图关联
  • View（V）视图层。管理HTML模板和视图渲染
  • Controller逻辑层。管理事件逻辑

前端渲染 View需要数据调用controller控制器中对应的方法，从Model中调用方法获取数据，返回给控制器再render到视图中。

手写实现MVC

// 利用MVC模式写一个计算器，具有加减乘除的功能
// 模块化
/**
 * Model:       1. 储存数据  2.绑定数据和视图关联关系
 * View:        1.html模板  2.渲染模板
 * Controller:  1.管理事件逻辑
 */
(function () {
    const Model = {
        data: {
            a: 0,
            b: 0,
            r: 0,
            symbol: '+',
        },
        init: function () {
            const _that = this;
            for (const key in _that.data) {
                Object.defineProperty(Model, key, {
                    get() {
                        return _that.data[key];
                    },
                    set(val) {
                        _that.data[key] = val;

                        View.render({[key]: val});
                    }
                })
            }
        }
    }

    const View = {
        el: 'root',
        template: `
                <div>
                    <input class="cal a"/> 
                </div>
                <div>
                    <input class="cal b"/> 
                </div>
                <div>
                    <span class="res-a">{{a}}</span> 
                    <span class="res-symbol">{{symbol}}</span> 
                    <span class="res-b">{{b}}</span> = 
                    <span class="res-r">{{r}}</span>
                </div>
                <div class="btn-group">
                    <button>+</button>
                    <button>-</button>
                    <button>*</button>
                    <button>/</button>
                </div>
        `,
        render(mutedData) {
            if (mutedData) {
                for (const key in mutedData) {
                    document.querySelector(`.res-${key}`).innerHTML = mutedData[key];
                }
            } else {
                this.template = this.template.replace(
                    /\{\{(.*?)\}\}/g,
                    function (node, key) {
                        return Model[key.trim()];
                    }
                );

                const div = document.createElement('div');
                div.innerHTML = this.template;
                document.getElementById(this.el).appendChild(div);
            }
        }
    }

    const Controller = {
        init: function () {
            document.querySelectorAll('.cal').forEach(item => {
                addEventListener('input', function (e) {
                    const key = e.target.className.split(' ')[1];
                    Model[key] = e.target.value;
                    Model.r =  eval('Number(Model.a)' + Model.symbol + 'Number(Model.b)');
                }, false);
            });

            document.getElementsByClassName('btn-group')[0].addEventListener('click', function (e) {
                Model.symbol = e.target.textContent;
                Model.r =  eval('Number(Model.a)' + Model.symbol + 'Number(Model.b)');
            });
        }
    }

    function init() {
        Model.init();
        View.render();
        Controller.init();
    }

    init();

})();

认识Mustache与Vue编译

模板语法

template内的HTML字符串，存在vue特性，比如文本、表达式、属性、指令。 Vue的模板都是基于HTML，模板中直接写HTML都是可以被HTML解析器解析的。 Vue提供了一套模板编译系统，可以将HTML字符串转换为AST树，然后再形成虚拟节点，进行渲染。

插值表达式

{{}}为插值表达式。

import { createApp } from 'vue'
const App = {
    data(){
        return {
            title: 'this is vue program'
        }
    },
    template:`
      <div>
        <p>{{title}}</p>
      </div>
    `
}
createApp(App).mount('#app')

Mustache解析插值表达式库

Mustache库提供了用更少的逻辑处理模板渲染的能力，GitHub地址 vue在处理插值表达式时参考了其实现，但是内部没有使用该库，vue具有自己的模板编译系统。

使用Mustache库

import Mustache from 'mustache';

var data = {
    title: 'this is vue program',
    a: '1'
}

var html = Mustache.render(`<p>{{title}}</p>`, data);

document.getElementById('app').innerHTML = html;

指令概念

directive指令：在所有vue中，模板上的属性v-都是指令。 vue提供了大量的内置指令，v-if、v-else、v-for、v-show、v-html、v-once。 同时开发者也可以自定义指令，命名规范：v-name。

  • 为什么叫做指令？
    • 指令规定了模板应该按照怎样的逻辑进行渲染或绑定行为，

v-once

  • 作用：一次插值，永不更新。使用 v-once 后会影响整个节点，包括其子节点中的内容也不会被更新。
// 当msg/title发生变化时，插值处的内容不会更新。
<p v-once>
    这个将不会改变: {{ msg }}
    <span>{{title}}</span>
</p>
  • 对于不可变的值，考虑不用v-once指令，可以使用全局变量并使用模板字符串语法。
import {createApp} from 'vue/dist/vue.esm-bundler'
const immutable = 'Test';

const App = {
    template: `
      <div>
        ${immutable}
      </div>
    `
}
createApp(App).mount('#app');

v-html

  • 作用：插值表达式不会解析HTML，会将数据解析为普通文本。若需要输出真实的HTML，需要使用v-html指令。
  • 不要试图用v-html做子模板，因为在v-html中写入的指令，vue不会进行解析。
  • 为什么插值表达式不会解析HTML? 由于插值表达式是js表达式，没有对DOM的操作，所以不会解析为HTML。
<p>Using mustaches: {{ rawHtml }}</p>
<p>Using v-html directive: <span v-html="rawHtml"></span></p>

v-html 和 xss攻击

不要把用户提供的值直接作为v-html的内容，这种插值可能会引起XSS攻击。虽然HTML5防御了部分常见的xss攻击，但是并不完全。

  • input输入<script>alert('1')</script>不会生效，html5做了优化。
  • <img src="/12" onerror="">依然可以进行xss攻击。
import {createApp} from 'vue/dist/vue.esm-bundler'

const App = {
    data() {
        return {
            inputVal: 'alert('1')'
        }
    },
    template: `
      <p>{{inputVal}}</p>
      <img src="/1231" :onerror="inputVal"/>
    `

}
createApp(App).mount('#app');
